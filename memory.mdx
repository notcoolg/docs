---
title: "Azura Memory"
---

Azura can remember important things about you across conversations. This **Long-term Memory** feature means Azura gets more personalized and contextually aware the longer you use it, without you having to re-explain yourself every session.

## How It Works

As you chat, Azura automatically identifies and stores facts that are likely to be useful in future conversations, things like your name, your preferred programming language, your job role, projects you're working on, or personal preferences you've mentioned.

These memories are stored securely in your account and are included as context whenever you start a new conversation.

### Examples of Things Azura Might Remember

- _"User prefers TypeScript over JavaScript"_
- _"User is working on a SaaS product called Horizon"_
- _"User likes concise answers without filler text"_
- _"User is learning Rust for the first time"_
- _"User's name is Alex"_

---

## Viewing Your Stored Memories

1. Open the **Settings** modal (click the settings icon in the header, or type `/settings`)
2. Navigate to the **Memory** tab
3. You'll see a list of all stored memories, up to a maximum of **100 entries**

---

## Deleting a Memory

To remove a single memory:

1. Open **Settings → Memory**
2. Hover over the memory you want to remove
3. Click the **delete (X)** button that appears on the right

The memory is deleted immediately and will no longer be used in future conversations.

---

## Clearing All Memories

To delete all stored memories at once:

1. Open **Settings → Memory**
2. Click **Purge All** in the top right of the memory list
3. Confirm the deletion in the dialog

> **Warning:** This action cannot be undone. All memories will be permanently deleted.

You can also use the `/forget` slash command from the Omnibox to clear all memories quickly without opening Settings.

---

## Disabling Long-Term Memory

If you'd rather Azura not save anything about you, you can disable memory entirely:

1. Open **Settings → Memory**
2. Click **Disable** in the Long-term Memory section

When disabled, Azura will stop saving new memories and existing memories will be "paused," they won't be used in new conversations. Your stored memories are preserved and can be reactivated later.

To re-enable: click **Enable** in the same settings panel.

> **Note:** The toggle state is saved locally per device/browser.

---

## Memory vs. Conversation Context

It's important to understand the difference:

|  | Long-term Memory | Conversation Context |
| --- | --- | --- |
| **Scope** | Persists across all conversations | Only applies to the current conversation |
| **Storage** | Saved in your account | Held in-memory during the session |
| **Examples** | Your name, preferences, ongoing projects | What you discussed earlier in this chat |
| **You control it** | Yes , view, edit, delete anytime | Cleared when the conversation ends |

---

## Privacy

- Memories are stored securely and associated with your user account
- They are only accessible to you and to Azura when generating responses
- Memories are never shared with other users
- You have full control to view and delete any memory at any time

---

## Tips

- **Not seeing personalization yet?** Give it a few conversations, Azura stores memories gradually as relevant facts come up.
- **Want Azura to remember something specific?** Just tell it directly: _"Remember that I prefer dark mode UI designs"_ or _"Note that I use Python 3.11"_.
- **Memory full?** You can store up to 100 memories. If you're near the limit, prune outdated ones in Settings.
- **Starting a private session?** Disable memory before chatting if you don't want the conversation to influence future context.
