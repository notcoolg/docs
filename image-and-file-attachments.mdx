title: "Image & File Attachments"
---

Azura supports attaching images and files to your messages. This lets you show Azura what you're working on, whether it's a screenshot, a diagram, a document, or a data file.

## Image Input (Vision)

Azura can look at images and answer questions about them. This feature is available exclusively with **Azura 1-Plus**.

### What you can do with images

- **Analyze screenshots**, ask Azura to explain UI elements, find bugs in designs, or read on-screen text
- **Interpret diagrams** — upload architecture diagrams, flowcharts, or whiteboard photos for Azura to describe or improve
- **Read charts and data** — send a graph or table and ask Azura to explain the trends or extract numbers
- **Debug visually** — paste a screenshot of an error or unexpected behavior and ask what's wrong
- **Design feedback** — get suggestions on UI layout, color choices, or visual hierarchy
- **Handwriting recognition** — upload a photo of handwritten notes for Azura to transcribe

### How to attach an image

**Option 1: Via the **`+`** menu:**

1. Click the `+` button on the left of the Omnibox
2. Select **Add Image** from the quick actions
3. Choose an image file from your device
4. Add your message and send

**Option 2: Paste from clipboard:**

- Copy an image (e.g., from a screenshot tool) and paste it directly into the Omnibox

### Supported image formats

- PNG
- JPEG / JPG
- GIF (first frame)
- WebP

### Important notes

- When an image is attached, Azura **automatically locks to Azura 1-Plus** for that conversation, since vision capability requires the Plus model
- Images are estimated at approximately **1,000 tokens** each for usage tracking purposes
- Very large images may be resized automatically

## File Attachments

You can attach text-based files to provide Azura with content to read and reference.

### Supported file types

- Text files (`.txt`)
- Code files (`.js`, `.ts`, `.py`, `.java`, `.go`, `.rb`, etc.)
- Markdown (`.md`)
- CSV and data files (`.csv`, `.json`)
- HTML/CSS (`.html`, `.css`)

### How to attach a file

1. Click the `+` button in the Omnibox
2. Select **Attach File**
3. Choose a file from your device
4. Add a message explaining what you'd like Azura to do with it
5. Press Enter or click Send

### What you can do with attached files

- _"Review this Python script for bugs and suggest improvements"_
- _"Summarize this document"_
- _"Convert this JSON to a TypeScript interface"_
- _"Explain what this code does"_
- _"Identify the columns in this CSV and write a pandas script to analyze them"_

## Adding a Code Snippet

If you want to share a short block of code directly in a message, use the **Code Snippet** action:

1. Click the `+` button in the Omnibox
2. Select **Code Snippet**
3. Paste or type your code
4. Add your question and send

Code snippets are formatted automatically in the message for better readability.

---

## Tips

- **Be specific:** When sharing an image, describe what you're looking for. "What's wrong with this design?" gets better results than no question at all.
- **Multiple images:** You can attach more than one image to a message if you need Azura to compare them.
- **Large files:** Very large files may be truncated. For best results, share the relevant section rather than an entire codebase.
- **Sensitive content:** Avoid sharing images or files containing passwords, API keys, or personal information you wouldn't want stored.
